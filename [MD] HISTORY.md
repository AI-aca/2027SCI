- **스코프 꼬임(Scope Issue) 버그 픽스**: 기존 코드에서 '수정 모드' 상태를 추적하는 `isEditMode` 변수가 `DOMContentLoaded` 이벤트 리스너 내부에 `let`으로 지역 변수로 선언되어 있었습니다. 반면, 톱니바퀴 [수정] 버튼을 누를 때 호출되는 `openEditStudent` 함수는 전역 함수라서, 수정 버튼을 누르면 '전역 공간'에 새로운 `isEditMode = true`를 억지로 생성하고 있었습니다. 
- 그 결과, [수정 완료] 버튼을 눌렀을 때 실행되는 클릭 이벤트는 전역 변수가 아닌 항상 `false`인 '지역 변수'만 바라보게 되어, 무조건 신규 등록 로직(saveStudent)을 타면서 "이미 등록된 연락처" 에러를 뿜어내는 상태였습니다.
- **해결 조치**: `isEditMode`를 완전히 파일 최상단 전역 스코프(Global Scope)로 끄집어내어 명시적으로 선언(`let isEditMode = false;`)하고, 내부의 지역 선언(`let `)을 제거하여 [수정] 버튼과 [등록 완료] 버튼이 완벽하게 동일한 상태값을 공유하도록 픽스했습니다. 이제 정상적으로 `updateStudent` (수정 통신) 로직을 타게 됩니다.

- **해결 방안**: Code.gs의 extractIdFromUrl 함수 내부에 /folders/ 문자열 파싱 정규식 조건을 추가하여, 폴더 URL에서도 정상적으로 ID만 추출할 수 있도록 보완 완료.
