## 1. 챗봇의 역할 (단 하나)

> **이 사이트를 처음 쓰는 사람이 "이거 어떻게 써요?"라고 물어봤을 때 답해주는 'UI 사용법 안내 가이드'**

- 입시 컨설팅 AI가 **아닙니다.**
- 채점 분석 엔진이 **아닙니다.**
- 개발자 기술 문서 검색기가 **아닙니다.**

---

## 2. 챗봇 접근 권한

| 사용자 | 챗봇 사용 가능 여부 |
|:---|:---:|
| 학생 (개별 링크 접속) | ✅ 사용 가능 |
| 교사 (비밀번호 로그인) | ✅ 사용 가능 |
| 관리자 (비밀번호 로그인) | ✅ 사용 가능 |

- **모든 사용자**가 동일한 챗봇 창을 사용합니다.
- 학생에게 챗봇 버튼을 숨기거나 별도 분리하지 **않습니다.**

- **공통 챗봇 버튼 및 창 겹침 방지**: 학생으로 로그인 시 화면 전체를 덮는 거대한 '자소서 작성 모달 창'이 열립니다. 챗봇 호출 버튼과 챗봇 대화창은 이 모달 창 뒤에 숨겨져 클릭이 불가능해지는 현상을 막기 위해, 시스템 내의 **모든 UI 요소(모달 포함)보다 무조건 최상단(z-index: 9999 등)에 위치하도록 강제**해야 합니다.

  - 배려, 나눔, 협력, 타인 존중, 갈등 극복 등 공동체/인성 관련 사례의 구체적 과정과 배우고 느낀 점을 파고드는 질문 (3개)

[제4조건 : 부족 시 보완]
- 자소서 작성 분량이 부족하여 30개를 채우지 못할 경우에만, 자소서에 기술된 수학 및 과학 탐구 활동의 개념 재검증 및 심화 응용 질문을 추가하여 반드시 30개를 맞추십시오.
