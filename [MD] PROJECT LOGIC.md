# 🧬 [2027 과학고 관리 사이트] 시스템 데이터 흐름 및 논리 구조 명세서 ([MD] PROJECT LOGIC.md)

## 1. 종합 시스템 아키텍처 및 데이터 스토어 구조

본 시스템은 기밀 입시 데이터의 유실 방지 및 입시 포털의 로딩 퍼포먼스를 확보하기 위해 **멀티 데이터 스토어 아키텍처(Multi-Store Architecture)**를 채택하고 있습니다.

```
                  ┌──────────────────────────────────────────┐
                  │          프론트엔드 브라우저 UI          │
                  │   (index.html, script.js, style.css)     │
                  └────────────────────┬─────────────────────┘
                                       │
                                       │ (HTTPS / Google Apps Script API)
                                       ▼
                  ┌──────────────────────────────────────────┐
                  │        백엔드 코어 (Code.gs)             │
                  │   - API Client 라우터 & 권한 검증       │
                  │   - Gemini AI & Advanced OCR 제어        │
                  └──────┬─────────────┬─────────────┬───────┘
                         │             │             │
        ┌────────────────┘             │             └────────────────┐
        ▼                              ▼                              ▼
┌───────────────┐              ┌───────────────┐              ┌───────────────┐
│  메인 시트 DB │              │ AI 산출근거   │              │ 학생 개별     │
│  (Main Sheet) │              │ 외부 시트 DB  │              │ 자소서 시트   │
└───────────────┘              └───────────────┘              └───────────────┘
 - 인적사항 & 합불              - 23개 상세점수                 - 탭 1: 최신본
 - 총점 (10열, J열)             - AI 리포트 전문                - 탭 2: 수기이력
 - PDF 링크 (9열)                                              - 탭 3: AI피드백
 - 자소서 상태 (13열)                                          - 탭 4: AI질문
```

### 2.2. 외부 AI 산출근거 데이터베이스 스키마 (`[생기부 산정근거]` 시트)
Gemini가 생성한 대용량 분석 데이터의 로딩 렉을 원천 차단하기 위해 실시간 분리 기록되는 보관함 스키마입니다.

- **열 1 (A)**: `timestamp` (평가 일시, `yyyy-MM-dd HH:mm:ss`)
- **열 2 (B)**: `studentPhone` (학생 식별 전화번호 또는 토큰)
- **열 3 (C)**: `name` (학생 이름)
- **열 4 (D)**: `targetSchool` (지원 고교)
- **열 5 (E)**: `totalScore` (총점, 0~400)
- **열 6 (F)**: `analysisReport` (Gemini 리포트 한글 원문 텍스트 전문)
- **열 7 (G)**: `scoreDetails` (29개 항목 정량/정성 평가 점수 및 근거 텍스트 JSON 블록)

## 3. 핵심 비즈니스 로직 및 데이터 파이프라인 (초정밀 분석)

본 섹션은 시스템 기동부터 버튼 클릭, AI 통신, 데이터 I/O에 이르는 전 과정을 소스코드 레벨로 정밀 해체한 명세서입니다.

### 3.1. [이벤트 0] 시스템 기동 및 로그인 시퀀스
- **기동 (DOMContentLoaded)**: 브라우저가 `index.html`을 로드하면 `script.js`의 DOMContentLoaded 리스너가 발동하여 초기 UI 이벤트 바인딩(사이드바 클릭, 모달 닫기 등)을 수행합니다.
- **게스트 차단 (무인증 블라인드 렌더링 강제)**:
  - 로그인 모달(ID: `login-modal`)이 화면을 덮습니다. 취소/닫기 버튼과 오버레이 클릭 시 닫히는 기능이 물리적으로 제거되어 있어 로그인을 강제합니다.
  - 이 상태에서 전역 변수 `CURRENT_ROLE`은 `null`이며, 대시보드의 주요 컨테이너(`main-content`, `sidebar`)는 CSS `display: none`이 강제 적용되어 백그라운드 데이터 유출을 원천 차단합니다.
- **인증 및 초기화 (`login()` 호출)**:
  - 사용자가 암호를 입력하고 `login()` 함수가 기동되면 백엔드의 `verifyPassword` API를 호출합니다.
  - 리턴된 `role` 값(관리자, 교사, 학생)이 `CURRENT_ROLE` 전역 변수에 바인딩되며, 모달이 닫히고 `display: block`으로 화면이 개방됩니다.
  - 직후 `loadSheetData()`가 비동기로 호출되어 메인 시트 데이터를 가져와 `originalData` 배열에 캐싱하고 `renderDynamicTableHeaders()`를 발동시킵니다.

### 3.2. [이벤트 1] 사이드바 네비게이션 및 동적 렌더링 로직
- **메뉴 스위칭 (`switchMenu(menuId)`)**:
  - 사이드바의 7대 메뉴(`menu-dashboard`, `menu-student-info`, `menu-record`, `menu-personal-statement`, `menu-interview`, `menu-user-guide`, `menu-settings`) 중 하나를 클릭하면 `switchMenu` 함수가 트리거됩니다.
  - 선택된 탭에 `active` 클래스가 부여되며, `currentMenu` 전역 변수가 갱신됩니다.
- **테이블 헤더 동적 교체 (`renderDynamicTableHeaders()`)**:
  - `updateTableForMenu()`가 연계 호출되며, `currentMenu` 값에 따라 고정 열 4개(센터명, 학생명, 학교, 지원학교) 우측에 붙을 동적 헤더들을 `thead`에 주입합니다. (예: 생활기록부 탭 클릭 시 '생기부', '생기부 점수(AI)' 헤더 생성)
- **데이터 필터 및 바인딩 (`renderTable(data)`)**:
  - 선택된 메뉴에 맞는 데이터만 필터링한 후, `tbody`에 `<tr>` 요소들을 동적으로 생성하여 주입합니다. 권한이 '학생'인 경우 타 학생의 데이터 렌더링을 DOM 레벨에서 드롭시킵니다.

### 3.3. [이벤트 2] 대시보드 버튼 인터랙션 명세
- **수정 버튼 (학생정보 탭)**:
  - `btn-action edit` 클릭 시 `openStudentEditModal(student)` 함수가 발동하여, 숨겨진 `modal-student-edit` 창을 띄우고 해당 학생의 전화번호, 강사명 등 기존 데이터를 input value에 바인딩합니다.
- **PDF 업로드 / 보기 버튼 (생활기록부 탭)**:
  - `btn-action upload` 클릭 시 숨겨진 `file` 인풋을 트리거하여 PDF를 선택받고, `uploadStudentRecordPdf` API를 통해 백엔드로 Base64 전송 후 뷰어 링크를 받아옵니다. 성공 시 버튼은 즉각 `보기 🔗`(`btn-secondary`)로 DOM 교체됩니다.
- **생기부 상세 모달 창 호출 (별도 팝업창)**:
  - 점수 텍스트(예: 285) 클릭 시, 테이블 아래로 열리는 구형 아코디언 방식이 아닌 **별도의 상세 모달 팝업 창(`modal-score-details`)**이 화면 중앙에 렌더링됩니다. (코드 상 일부 클래스명에만 accordion이라는 낡은 네이밍이 남아있을 뿐, 실제 렌더링은 완벽한 모달창 형태를 갖춥니다.)
  - 클릭과 동시에 백엔드의 `getScoreDetailsBasis` API를 비동기 호출합니다.
  - 리턴받은 29개 항목 JSON 데이터를 모달 내의 `#score-details-grid` DOM 내부에 23개의 글로우 카드로 동적 생성하여 주입합니다.
- **자소서 작성 모달 창 호출 (자기소개서 탭)**:
  - 버튼 클릭 시 `loadStudentModal(studentPhone)`이 발동하여 거대한 `modal-student-details` 창을 엽니다.
  - 내부적으로 `renderStudentInfo()`, `renderPersonalStatementTabs()`, `getPersonalStatementHistory()`가 동시다발적으로 기동되어 좌우 이원화 탭과 이력 드롭다운을 동기화합니다.
- **일괄 AI 실행 버튼 (설정 탭)**:
  - 관리자가 일괄 버튼 클릭 시, 프론트에서 최종제출 상태인 학생 배열을 필터링하여 루프를 돌며 백엔드 API를 순차 호출합니다.
  - 프로그래스바(`showProgressBar`, `updateProgressBar`)가 DOM에 마운트되어 (현재 완료 카운트 / 전체 대상 수) * 100 공식으로 `width %`를 시각적으로 채워나갑니다.

### 3.4. [이벤트 3] AI 엔진 연동 파이프라인 (Double-Layer)
- **생기부 정밀 채점 트랜잭션**:
  1. PDF 업로드 직후 백엔드의 `evaluateStudentRecord` 발동.
  2. 구글 Advanced Drive Service(`extractTextFromPdf`)가 임시 Docs를 생성해 OCR 텍스트를 파싱.
  3. Gemini 1.5 Flash/Pro 엔진으로 텍스트+프롬프트 전송, 29개 항목 JSON 획득.
  4. `calculateRecordScore(scoreJson)` 함수로 400점 만점 연산 후 J열 총점 갱신.
  5. **대용량 데이터 격리**: 산정 근거 텍스트와 23개 JSON 블록은 무거운 메인 시트 대신 `saveBasisToExternalSheet()`를 통해 외부 시트(`1pKG3j...`)에 실시간 분리 기록됩니다.
- **자소서 첨삭 및 질문 생성 파이프라인**:
  - `generateAIFeedback`, `generateAIQuestions` API 호출 시 Gemini가 응답을 생성하며, 생성된 텍스트는 해당 학생의 개별 자소서 스프레드시트 내 `[AI 피드백 이력]`, `[AI 예상질문]` 탭에 즉시 영구 저장됩니다.

### 3.5. [이벤트 4] 자소서 입출력(Read/Write) 롤백 및 락(Lock) 무결성 매커니즘
- **Dirty Check (무수정 방지 모듈)**:
  - `btn-save-ps` (임시 저장) 클릭 시, `savePersonalStatement` 로직 내부에서 기존에 메모리에 로드된 `originalContents`와 에디터의 현재 `value`를 `===` 연산자로 비교합니다. 변화가 감지된 문항의 배열만 추려내어 네트워크 페이로드로 전송해 서버 부하를 막습니다.
- **이원화 롤백 엔진**:
  - 백엔드는 개별 학생 시트의 `최신 자소서` 탭에 값을 덮어쓰는 동시에, `작성 이력` 탭의 최하단에 타임스탬프와 함께 백업 행을 추가(Append)합니다.
  - 프론트에서 이력 드롭다운(`ps-version-selector`)의 옵션 변경 시 `change` 이벤트가 발동하여, 메모리에 받아둔 과거 텍스트를 `textarea`에 즉시 뿌려주어 시각적 롤백을 구현합니다. (저장 버튼을 누르는 순간 롤백이 확정됨)
- **학생 주도 마감 및 강제 락(Lock) 체계**:
  - 학생이 `[🚀 자소서 최종 제출하기]` 버튼을 누르면, 백엔드가 메인 시트 상태를 `최종제출`로 고정합니다.
  - 프론트는 이 상태를 감지하여 텍스트 에디터에 `readOnly = true` 속성을 부여하고, 저장/제출 버튼 DOM을 `display: none`으로 완전히 날려버립니다.
  - **반려(Unlock) 루틴**: 오직 관리자 권한 로그인 시에만 모달창 하단에 `[🔓 최종 제출 반려]` 버튼이 렌더링되며, 클릭 시 관리자 암호를 한 번 더 검증(`verifyAdminPasswordForUnlock`)한 뒤 락을 해제합니다.


### 3.6. [이벤트 5] 4단계 권한 계층 및 프론트-백 이중 방어 체계 (보안 로직)
부록에 정의된 기능별 권한 매트릭스(O/X표)를 시스템 내부적으로 완벽하게 통제하기 위해, 해킹 방어를 염두에 둔 '이중 방어(Double-Layer Validation)' 파이프라인이 기동됩니다.

- **권한 부여 동적 트랜잭션 (`verifyPassword` API)**:
  - 백엔드는 패스워드를 하드코딩하지 않고, `getPasswordsFromSheet()`를 통해 구글 스프레드시트의 `[설정]` 탭에서 실시간으로 Admin과 User 암호를 긁어옵니다. (시트가 누락된 경우에만 기본값 폴백 가동)
  - 암호 대조 후 `role: '관리자'`, `role: '교사'`, `role: '미인증 사용자'` 중 하나의 상태를 프론트엔드로 리턴하며, 특정 학생 전용 링크 토큰으로 접속 시 `role: '학생'`이 강제 부여되어 프론트엔드 전역 변수 `CURRENT_ROLE`에 고정됩니다.
- **[1차 방어] 프론트엔드 UI 렌더링 물리적 차단 (DOM Drop)**:
  - `CURRENT_ROLE`이 `'학생'` 이거나 `'미인증 사용자'`일 경우, `script.js`에서 테이블이나 모달창의 DOM 요소를 그릴 때(예: `btn-action` 등 각종 제어 버튼, `tab-btn-ai-feedback` 등 기밀 탭) 해당 요소의 HTML 태그 문자열 렌더링 자체를 `if`문으로 스킵합니다. 
  - 단순히 CSS로 숨기는 것이 아니라 DOM 트리에 물리적으로 생성되지 않게 잘라내어, 악의적인 사용자가 개발자 도구를 열어 버튼을 활성화하는 것을 원천 차단합니다.
- **[2차 방어] 백엔드 API 패킷 재검증 (API-Level Validation)**:
  - 1차 방어를 뚫고 개발자 콘솔 등을 통해 해커가 강제로 백엔드 API(REST)를 직접 호출하는 공격을 방어합니다.
  - `getScoreDetailsBasis(studentPhone, clientRole)` 등 기밀 데이터를 요청하는 API의 진입점에 도달하면, 백엔드 로직이 즉각 파라미터로 넘어온 `clientRole`을 재검증하여 `if (clientRole === '학생' || clientRole === '미인증 사용자') return null;` 코드로 트랜잭션을 그 즉시 튕겨냅니다.
  - 자소서 수정 반려(`unlockPersonalStatement`) 등 관리자 고유의 파괴적 기능 발동 시에는 단순 Role 문자열 체크에 그치지 않고, **관리자 패스워드 본체를 한 번 더 요구하여 검증(`verifyAdminPasswordForUnlock`)**하는 극단적인 철통 보안 체계를 가동합니다.

## 4. UI/UX 디자인 시스템 로직 (사이버네틱 퓨처 테크)

본 프로젝트는 미래지향적인 입시 관리 시스템의 정체성을 시각적으로 확립하기 위해 **'사이버네틱 퓨처 테크(Cybernetic Future Tech)'** 디자인 테마를 전역적으로 강제합니다. 향후 어떠한 프론트엔드 기능 추가 시에도 아래의 디자인 규칙을 절대 준수해야 합니다.

### 4.1. 절대 금지 사항 (Strict Prohibitions)
- **Tailwind CSS 사용 절대 금지**: 모든 스타일링은 Vanilla CSS (`style.css`)의 사전에 정의된 `var(--)` 변수 통제하에 작성해야 합니다.
- **rem 단위 사용 절대 금지 (규칙 #0-7 연계)**: 기기 및 브라우저의 기본 폰트 설정 변화에 의한 레이아웃 붕괴를 원천 차단하기 위해, 모든 글꼴(`font-size`), 마진(`margin`), 패딩(`padding`) 등 렌더링에 영향을 주는 수치는 **무조건 고정 px 단위**만 사용해야 합니다.

### 4.2. 컬러 스키마 및 테마 변수 (Color Schema)
`style.css`의 `:root`에 정의된 색상 변수를 통해서만 색을 지정합니다.
- **배경 (Backgrounds)**:
  - `--bg-primary`: `#0f172a` (Slate 900) - 메인 베이스
  - `--bg-secondary`: `#1e293b` (Slate 800) - 카드, 모달 등 패널 베이스
  - `--bg-tertiary`: `#334155` (Slate 700) - 호버 액션, 입력창 등
- **포인트 컬러 (Accents)**:
  - `--color-primary`: `#10b981` (Emerald 500) - 긍정, 성공, 실행 버튼 (네온 글로우 주축)
  - `--color-secondary`: `#0ea5e9` (Sky 500) - 정보, 링크, 보조 액션
  - `--color-warning`: `#f59e0b` (Amber 500)
  - `--color-danger`: `#ef4444` (Red 500)

### 4.3. 핵심 시각 효과 (Visual Effects)
- **네온 글로우 (Neon Glow)**:
  - `--neon-glow-cyan`: `0 0 10px rgba(6, 182, 212, 0.5)`
  - `--neon-glow-emerald`: `0 0 10px rgba(16, 185, 129, 0.5)`
  - 주로 버튼 호버(`:hover`) 시 발동하여 사이버네틱한 상호작용 피드백을 제공합니다.
- **글래스모피즘 (Glassmorphism)**:
  - 모달 오버레이 및 상단 헤더 영역 등에 `backdrop-filter: blur(15px) saturate(150%)`를 적용하여 뒷배경이 은은하게 비치는 미래적 유리 질감을 구현합니다.
  - 모달 컨테이너의 테두리는 `border: 1px solid rgba(255, 255, 255, 0.1)` 로 얇게 둘러 빛 반사 효과를 모사합니다.
- **Nebula Radial Gradient (성운 효과)**:
  - `body` 배경에 2중 `radial-gradient`를 깔아 심연의 우주 같은 입체적 배경 뎁스를 만듭니다.

## 4. 챗봇(SION) 지식 체계 및 프롬프트 로직

챗봇 `SION`은 학생을 포함한 모든 사용자가 시스템 사용 안내와 자소서 가이드를 편리하게 받을 수 있도록 단일 통합 대화창으로 운영됩니다. 단, 학생이 챗봇을 사용하여 입시 기밀을 탈취하는 보안 붕괴 사고를 원천 차단하기 위해 지식 범위를 극도로 제한합니다.

- **채점 및 입시 기밀 정보의 100% 영구 배제**
  - **제거 정보**: 23개 생기부 채점 공식, 감점 룰(성취도 B/C 감점, 출결 감점 수치 등), 합격선 컷오프 점수(300점, 275점, 220점), 실제 모의 평가 합불 사례(김수인 343점, 김주찬 사배자 282점 탈락 등) 등 AI 채점 및 분석에 관련된 모든 데이터는 챗봇 프롬프트에서 단 한 글자도 남기지 않고 전수 배제합니다.
- **오직 시스템 기능 사용법 가이드만 제공**
  - 챗봇은 오직 사용 설명서에 명시된 기능적인 방법(자소서 작성창 글자 수 세기 기준, 버전 롤백 복원 방법, 임시저장 방법, 최종 제출 후 수정 불가 알림, PC 크롬 권장 사양 등)에 대해서만 인지하여 대답하도록 설계합니다.
  - 학생이 우회 질문이나 입시/채점 정보를 유도하여 물을 경우: "해당 정보는 보안 정책상 안내가 불가합니다." 또는 "기능 가이드 외의 입시 정보는 대답할 수 없습니다."로 철저히 예외 방어하도록 프롬프트 지침을 고정합니다.

## 5. 권한 및 보안 매커니즘

- **이중 보안 검증 (Double-Layer Validation)**
  - **1차 UI 레이어**: `CURRENT_ROLE`이 `'학생'` 또는 `'미인증 사용자'` 일 경우 상세 뷰어 아코디언 컴포넌트의 클릭 액션을 이벤트 리스너 차원에서 물리 차단합니다.
  - **2차 API 레이어**: 비인가된 사용자가 스크립트 콘솔 등을 이용하여 `getScoreDetailsBasis`나 AI 실행 함수를 강제 요청하는 해킹 방지를 위해 백엔드 API 단에서 `clientRole` 파라미터를 강제 검증하고 즉각 Reject 처리합니다.
- **학생 접속 한도 및 배너 통제**
  - 자소서 모달에 학생용 링크로 접근 시, `tab-btn-ai-feedback` 등 AI 분석 관련 탭은 웹 페이지 DOM 구조 상에서 아예 렌더링되지 않으며, 화면 최상단에는 크롬 사용 권장 및 로그아웃 유도를 위한 `student-warning-banner`가 상시 노출되어 입시 기밀의 누설을 이중 차단합니다.

## 4. 프론트엔드 모달 창 동작 및 권한 제어

### 4.1. 자소서 모달 창 내 버전 관리 및 롤백
- **독립적 제어**: 자소서 이력 드롭다운(좌측)과 피드백 이력 드롭다운(우측)이 서로 구분되어 있어, 각각 개별적으로 과거 버전을 불러오거나 롤백(복원)할 수 있다.
- **최신본 강조**: 각 드롭다운 리스트 내 최상단에는 항상 최신 버전이 강조 표시(⭐ 마킹 등)되어 노출되어야 한다.
- **비동기 통신**: 저장 및 히스토리 불러오기 시 화면 전체가 깜빡이거나 새로고침되는 것을 금지하며, 비동기(AJAX)로 텍스트만 실시간 교체 적용한다.

### 4.2. 학생 주도 최종 제출 및 잠금(Lock) 규칙

## 3. 자기소개서 작성 및 최종 제출 통제 프로세스

자소서의 편집과 마감은 무결한 비즈니스 룰에 의해 안전하게 격리 및 통제됩니다.

### 3.1. 로컬 오토세이브(0.1초 자동 저장) 및 Dirty Check 일괄 저장
- **로컬 0.1초 자동 저장**: 자소서 및 교사 피드백 작성 시, 사용자가 글자를 입력할 때마다 `oninput` 이벤트를 통해 브라우저의 전역 메모리(`currentPsAnswers` 등)에 0.1초 단위로 즉각 자동 저장됩니다. 이를 통해 문항 드롭다운 탭을 이동하더라도 작성 중인 데이터가 절대 날아가지 않는 완벽한 UX를 제공합니다.
- **Dirty Check 최종 일괄 전송**: 모달을 닫거나 [임시 저장하기]를 클릭할 때, 프론트엔드가 원본(AS-IS) 텍스트와 메모리의 현재 입력값(TO-BE)을 문항별로 비교합니다.
- 실제 변경 내용이 발생한 문항만 추출하여 Apps Script API에 패킷으로 단 한 번만 일괄 전송(`savePersonalStatement`)함으로써 데이터 전송 병목을 최소화하고 불필요한 버전 로그 무한 증식을 방지합니다.

### 3.2. 최종 제출 (Lock) 및 반려(Unlock) 워크플로우
- **최종 제출**: 학생이 작성을 마치고 [🚀 최종 제출하기]를 클릭하면, 메인 시트의 자소서 제출 상태 컬럼(M열)이 `최종제출`로 강제 고정됩니다.
- **수정 잠금**: 최종 제출이 완료되면 자소서 모달 상단에 경고 배지가 노출되며, 자소서 입력 영역(`textarea`)이 `readOnly`로 전환되고 [임시 저장] 및 [최종 제출] 단추가 완전히 비활성화됩니다.
- **반려 절차**: 관리자가 모달을 열면 `🔓 최종 제출 반려 및 락 해제` 단추가 활성화됩니다. 클릭 시 관리자용 검증 패스워드를 실시간 조회/검증하여 통과될 경우에 한해서만 상태를 `작성중`으로 원복하여 락을 해제합니다.

## 4. 이원화 버전 롤백 엔진

자소서와 수기 피드백은 개별 스프레드시트 내에서 `최신 자소서`와 `작성 이력` 탭으로 이원화 관리됩니다.

- **작성 이력 로깅**: 자소서나 피드백이 실시간 저장될 때마다 `작성 이력` 탭에 `[저장 일시, 작성자, 문항 번호, 구분(자소서/피드백), 저장 내용]`이 누적 로그 형태로 추가 보관됩니다.
- **이력 조회**: 편집창을 열면 Apps Script는 해당 학생의 `작성 이력` 탭을 전수 조회하여 해당 문항의 과거 이력을 시간 역순으로 드롭다운(`ps-version-selector`, `feedback-version-selector`)에 바인딩합니다.
- **최신본 강조 (⭐ 표기)**: 버전 드롭다운의 최상단에 `⭐ 최신 작성 내용` 옵션을 제공하여 현재의 활성 작업 버전을 즉시 인식하게 돕습니다.
- **독립 롤백 복원**: 과거 버전을 클릭할 경우 화면상의 에디터에 임시 복원됩니다. 사용자가 눈으로 팩트를 확인한 후 [임시 저장하기]를 누르면 해당 이력 본이 `최신 자소서` 탭에 덮어씌워지며 최신 활성본으로 안전하게 롤백이 종결됩니다.

생활기록부 총점(400점 만점)을 기준으로 다음과 같이 입시 지원 등급 컷오프 및 지원 전략 기준선을 정의합니다.

   - **중 (220점 ~ 274점)**: 일부 과목 감점이나 탐구 실적 부족으로 1차 서류 합격이 다소 불안정하므로 면접 및 자소서로 강한 역량 보완이 필요합니다.
   - **하 (220점 미만)**: 수학/과학 성취도 B 이하 감점이 누적되었거나 탐구 기록이 극도로 빈약하여 불합격 가능성이 높은 등급입니다. (현실적 지원 불가선: **220점 미만**). 실제 220점 미만(박은열 219점, 박재희 210점)은 예외 없이 전원 탈락하였습니다.

   - 사회통합전형 지원 대상자의 경우, 일반전형 대비 서류 심사 및 면접 통과를 위한 특별 검증 기준으로 생활기록부 평가 점수 **300점 이상** 획득을 합격 안전선으로 권장합니다. 실제 모의 평가 통계에 따르면 김주찬 학생이 282점이라는 초고득점을 기록했음에도 불구하고 사회통합전형에서 최종 불합격(탈락)하였습니다. 즉, 사배자는 일반 전형보다 문턱이 높아 최소 300점 이상이어야 안전하게 합격을 도모할 수 있다는 사실이 입증되었습니다.

## 6. 실시간 AI 채점 및 PDF 텍스트 추출 파이프라인 매뉴얼

생활기록부 점수 산출의 신뢰성을 보장하기 위해 PDF 텍스트를 기계적으로 추출하여 AI 채점 엔진에 직접 전달하는 전 과정은 아래의 흐름을 따릅니다.

### 6.1. PDF 파일 업로드 및 클라우드 동기화
- **업로드 실행**: 교사 또는 관리자는 대시보드의 해당 학생 행에서 [업로드] 단추를 통해 생활기록부 PDF 파일을 선택합니다.
- **클라우드 보관**: Apps Script API(`uploadStudentRecordPdf`)가 기동되어 파일을 Base64로 수신한 후, 구글 드라이브 지정 폴더(`STUDENT_RECORD_DIR`)에 즉시 저장하고 공유 권한을 '링크가 있는 모든 사용자'로 설정하여 뷰어 링크를 획득합니다. 이 링크는 대시보드 메인 시트의 9번째 열(`recordPdf`)에 자동으로 기입됩니다.

### 6.2. Advanced Drive OCR 기반 텍스트 추출
- **임시 구글 문서 변환**: 업로드 성공 시, 백엔드는 Advanced Drive Service를 기동하여 업로드된 PDF 파일을 Google Docs 파일 형식으로 복사/변환합니다. 이때 `ocr: true` 및 `ocrLanguage: 'ko'` 파라미터를 강제 적용하여 PDF 내의 인쇄 영역, 테이블, 이미지 텍스트를 고정밀 텍스트 데이터로 변환합니다.
- **본문 추출 및 삭제**: 변환된 임시 구글 문서 파일 ID를 활용하여 `DocumentApp.openById()`로 본문을 연 뒤, `getBody().getText()`로 텍스트 데이터를 통째로 스트링 변환하여 가져옵니다. 텍스트 추출이 완료된 즉시 구글 문서는 드라이브 휴지통으로 이동(`setTrashed(true)`)하여 드라이브 용량 증가를 막습니다.

### 6.3. Gemini AI 실시간 채점 및 결과 분리 보관
- **Gemini 채점 API 트리거**: 추출된 텍스트 스트링은 즉시 `evaluateStudentRecord` 함수의 `recordText` 인자로 전달되어 Gemini API로 송신됩니다.
- **채점 및 판정**: Gemini API는 내장된 `생기부분석프롬프트` 및 400점 채점 룰에 의거하여 29개 항목에 대한 수치와 팩트 기반 근거 텍스트를 추출하고, 최종 점수를 계산합니다.
- **이원화 저장**: 연산된 총점은 메인 시트 J열에 즉시 업데이트되고, 상세 채점 JSON 구조체(`scoreDetails`)와 AI 리포트 전문(`analysisReport`)은 외부 시트(`AI_SCORE_BASIS_SHEET`)의 `[생기부 산정근거]` 탭에 즉시 신규 행으로 누적되거나 덮어쓰여 저장됩니다.

## 7. 상세 채점 내역 불러오기 및 아코디언 연동 매뉴얼

- **데이터 로드**: 화면 갱신이나 새로고침 없이 프론트엔드는 백엔드 API인 `getScoreDetailsBasis(studentId, CURRENT_ROLE)`를 호출합니다.
- **외부 시트 쿼리**: 백엔드는 외부 시트의 `[생기부 산정근거]` 탭에서 해당 학생의 레코드를 쿼리하여 `scoreDetails` JSON과 `analysisReport` 텍스트를 패킷으로 구성해 프론트엔드로 리턴합니다.

### 7.2. 영역별 23개 세부 평가 카드 렌더링
- **동적 DOM 카드 빌드**: 프론트엔드는 전달받은 JSON 스키마를 판독하여, 23개 개별 채점 항목을 학업역량, 진로적합성, 인성 3대 영역으로 분류한 뒤 각각의 만점 대비 획득 점수, 판정 기준, 팩트 인용구(`quote`)가 기재된 글로우 카드를 생성하여 `score-details-grid`에 주입합니다.
- **종합 보고서 연동**: 우측 영역에는 AI가 작성한 종합 평가 리포트 텍스트 전문을 마크다운 포맷팅하여 배치하여 교사가 한눈에 학생의 생기부 장단점을 파악할 수 있도록 도웁니다.

### 7.3. 권한별 접근 차단 및 보안 배너 정책
- **접근 제한**: `CURRENT_ROLE`이 `'학생'` 또는 `'미인증 사용자'` 인 클라이언트가 아코디언 토글을 클릭할 시, 백엔드 및 프론트엔드 이중 보안 레이어에 의해 조회가 철저히 거부되며 *"보안 규정상 학생 계정은 생기부 상세 내역을 조회할 수 없습니다."* 경고 얼럿이 발생합니다.

```javascript
  else if (totalScore >= 215) resultBadge = '중하(서류탈락 위험)';
  else if (totalScore >= 190) resultBadge = '하(서류탈락)';
  else resultBadge = '최하(서류탈락)';

  let recommendation = '인천과학고 지원 권장';
  if (data.isSocialCare) {
    if (totalScore >= 300) {
      recommendation = '지원 가능 (사배자 전형 합격 안전권)';
    } else {
      recommendation = '지원 비권장 (사배자 전형 커트라인 300점 미만)';
    }
  } else {
    if (totalScore < 220) {
      recommendation = '지원 불가 (현실적 서류 합격 불가선)';
    } else if (totalScore < 275) {
      recommendation = '지원 비권장 (인천과고 권장선 275점 미만)';
    }
  }

  return {
    totalScore,
    areaScores: { area1, area2, area3 },
    itemScores: scores,
    resultBadge,
    recommendation
  };
}
```

- **가짜 [업로드] 알림창 (`alert`)**: `alert('[가상 업로드] ${file.name} (크기 정상 통과)...')` 
  - 백엔드 통신(Endpoint) 자체가 스크립트에 존재하지 않음.
- **가짜 PDF 목록 렌더링 (`innerHTML`)**: `pdfList.innerHTML = '<button class="pdf-item active">2027 인천과학고.pdf</button>'` 
  - 백엔드를 거치지 않고 JS 내에 하드코딩된 가짜 DOM 렌더링.

## 🚀 7. 스크립트 내부 동적 UI 텍스트 (`script.js` 내 하드코딩)
화면에 임시로 그려지는 텍스트의 소스코드 위치 맵핑입니다. 이 텍스트들은 백엔드와 무관하게 브라우저 렌더링단에서 생성됩니다.

- **권한 상태 텍스트 (`document.getElementById('current-role-display').textContent`)**:
  - `CURRENT_ROLE === '학생'` → "학생 전용 작성"
  - `CURRENT_ROLE === '교사'` → "일반 교사 계정"
  - `CURRENT_ROLE === '관리자'` → "시스템 관리자"
- **대시보드 검색창 (`document.getElementById('search-student')`)**:
  - `placeholder="학생명 또는 학교 검색..."`
- **로딩 오버레이 텍스트 (`document.getElementById('loader-text').textContent`)**:
  - `showGlobalLoader('문구')` 호출 시 전달되는 파라미터 텍스트 매핑.
- **일괄 작업 프로그레스바 (`document.getElementById('bulk-progress-text').innerHTML`)**:
  - `[${title}] 준비 중... (0/${total}명)`
- **AI 챗봇 초기 메시지 (`ai-chat-messages` 내 `.message.system`)**:
  - `"안녕하세요! 2027 과학고 합격을 위한 관리 사이트의 비서 SION입니다."`

`script.js` 내부에서 `alert()`, `confirm()`을 호출하는 정확한 트리거 이벤트와 그 원천입니다.
